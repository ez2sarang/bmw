Introduction

Package structure

PixelAdmin package has the next structure:

  • builder - Builder app
  • dist - Distribution / Compiled sources
    • css
      • themes
    • js
      • amd - RequireJS modules
    • demo - Demo assets
  • docs - Documentation
  • html - HTML files
    • static - Simple HTML
    • requirejs - RequireJS
  • js - Javascript sources
    • build - Babel working dir
    • src - ES6 sources
      • extensions - Extensions for external plugins
      • plugins - PixelAdmin plugins
  • libs - External libs / Plugins
  • scss - SCSS sources
    • mixins
    • plugins
    • themes
    • utitlilies
    • widgets
  • tasks - Gulp tasks

Custom build structure

Custom build (created with the Builder app) has the next structure:

  • custom - Custom files
    • build - Babel working dir
    • js - Custom JS sources, which can be written using ES6
    • scss - Custom SCSS sources
      • themes - Custom SCSS themes
  • js
    • build
    • src
      • extensions
      • plugins
  • scss
    • mixins
    • plugins
    • themes
    • utitlilies
    • widgets
  • tasks

After running the npm install, npm run bower:install and npm run compile commands, the custom build directory also will contain dist, libs and node_modules directories.
