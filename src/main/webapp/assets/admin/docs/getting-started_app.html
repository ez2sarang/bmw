Builder app

Builder app allows you easily create custom PixelAdmin bundles.

Before using Builder app you need to install Node.js and Git. See the Installation page.
builder directory is compressed into a .zip file. Uncompress the builder-password_is_1.zip file into the package directory. Password for the archive: 1

Supported platforms

Builder is available in three versions:

Windows (32-bit)

builder/Builder-win32-ia32/Builder.exe

Linux(Ubuntu) (64-bit)

builder/Builder-linux-x64/Builder

Mac OS X (64-bit)

builder/Builder-darwin-x64/Builder.app

Selecting destination path

If you want to create a new bundle, choose the destination folder and specify unique build directory.


You can choose an existing build directory path to update it.


You can use Builder app to update PixelAdmin version of the existing bundle. Sources in the custom directory will not be affected.

For example:

  1. You created a bundle using Builder app from PixelAdmin v2.2.0 package.
  2. When a new version, just download it and update the bundle using Builder app from the new package.

Selecting components

Select required components. You do not need to worry about dependencies - all dependencies between components will be handled automatically.

Configuring

The "Options" tab lets you configure build/compilation process. The available options:

  • Include dark version - check to include dark version of PixelAdmin (required for dark themes)
  • Compile sources - uncheck if you do not need compile sources
  • Compile AMD - uncheck if you do not need compile AMD modules
  • Minify AMD modules - check to minify AMD modules
  • Include RTL support - check to generate RTL version

Also, you can change the default dist directory path. This is useful if you need to compile sources to a directory that placed outside of the build directory.
Note: Dist directory path must be relative to the build directory path.


Warning!

Contents of the {dist-path}/css and {dist-path}/js directories will be erased before each compilation. So be careful changing the dist path.

For example, you have the next project structure:

  • site
    • assets
      • css
        • your-stylesheet.css
      • js
        • your-script.js
    • vendor
      • pixeladmin
    • ...

If you set the dist path equal to ../../assets, then the assets/css/your-stylesheet.css and assets/js/your-script.js files will be erased. To avoid this issue, just add to the dist path one more directory level: ../../assets/pixeladmin. Finally, the assets directory will have the next structure:

  • assets
    • css
      • your-stylesheet.css
    • js
      • your-script.js
    • pixeladmin
      • css
      • js

Selected options also will affect the Gulp tasks. For example:

  • if you uncheck "Compile sources" checkbox, then the compile / compile:* command will not compile SCSS/JS/AMD sources.
  • if you uncheck "Compile AMD" checkbox, then the compile / compile:* command will not compile AMD modules.

Building

Click on the start button and wait for the building process end.

Troubleshooting

  • On Mac OS X, when you trying to open Builder app, appears a window "“Builder” can't be opened because it is from an unidentified developer."

    See the solution at https://support.apple.com/kb/PH21769.

  • On Mac OS X El Capitan / Sierra you getting the "xcrun: error: invalid active developer path" error while installing Node / Bower packages

    See the solution at http://stackoverflow.com/questions/32893412/command-line-tools-not-working-os-x-el-capitan-macos-sierra/32894314#32894314.

  • Build process get stucked on the "Installing NPM packages" or "Installing Bower packages" step

    If the "Installing NPM packages" or "Installing Bower packages" step is in progress more than 10-15 minutes (according to speed of your internet connection), then there is a possibility that it's stucked. Most likely, the internet connection was interrupted. Do the next steps:
    1. Close the Builder window
    2. Check your internet connection
    3. If you have a firewall, check that Node, NPM and Git has access to the internet
    4. Open Builder app and try to run build process again
    If this is not solved the problem, try to install packages manually using terminal.

  • Builder app launched with message "Before continue, please install Nodejs", but Node.js installed

    On Windows:
    1. Open Control panel > System > Advanced system settings
    2. Click "Environment Variables..." button
    3. Find Path variable inside the "System variables" list
    4. Check that the variable contains path of the Node.js executable. For example, on x32 platforms, Path variable must contain C:\Program Files\nodejs\
    5. Logout from the system and login again

    On Linux(Ubuntu) and Mac OS X this issue can appear if you have installed Node.js using a Node Version Manager. To fix this issue, check that ~/.bash_profile(if exists), ~/.profile(if exists) or ~/.bashrc(if exists) file contains command that loads the Node Version Manager (See docs of the Node Version Manager which you're using).

    For example, if you're using nvm on Mac OS X, check that ~/.bash_profile file contains the next lines:

    export NVM_DIR="~/.nvm"
    [ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh" # This loads nvm

    Builder trying to load files in the next sequence:
    1. ~/.bash_profile
    2. ~/.profile
    3. ~/.bashrc
    Builder will load the first existing file. For example, if ~/.bash_profile doesn't exist, then Builder will try to load ~/.profile. If ~/.profile exists, then it will be loaded and existence of ~/.bashrc will not be checked.
